TUTORIAL — Solver SIMPLEX
=========================
Programa que resolve problemas de Programação Linear pelo método SIMPLEX
tabular (convenção Cj−Zj), com método das Duas Fases para restrições >= e =.

COMO EXECUTAR
-------------
1. Instalar o .NET SDK 10 (https://dotnet.microsoft.com/download).
2. Abrir um terminal na pasta do projeto.
3. Executar:  dotnet run
4. Seguir os passos da interface (entrada pelo teclado).

COMO INFORMAR OS DADOS
----------------------
Ao iniciar, o programa mostra um menu:
  [1-8] problemas de teste prontos (os mesmos do README);
  [9]   entrada manual.

Na entrada manual, cada linha descreve uma parte do problema:
1. Objetivo: "max" ou "min" seguido dos coeficientes c1..cn
   separados por espaço. Ex.: max 3 5
   (o número de variáveis é inferido da quantidade de coeficientes)
2. Restrições, uma por linha: os n coeficientes, o operador
   (<=, >= ou =) e o lado direito b. Ex.: 3 2 <= 18
3. Linha vazia (Enter) encerra a lista de restrições.

Aceita decimais com ponto ou vírgula (ex.: 1.5 ou 1,5).
Restrições com b negativo são normalizadas automaticamente
(multiplica-se a linha por −1 e inverte-se o operador).

Também é possível ler o problema de um arquivo via stdin:
    dotnet run < problema.txt

EXEMPLO RESOLVIDO (maximização)
-------------------------------
Problema:
    Max Z = 3x1 + 5x2
    s.a.   x1        <= 4
                2x2  <= 12
           3x1 + 2x2 <= 18
           x1, x2 >= 0

Entrada digitada (cada linha é um Enter):
    9              ← entrada manual
    max 3 5        ← objetivo
    1 0 <= 4       ← restrição 1
    0 2 <= 12      ← restrição 2
    3 2 <= 18      ← restrição 3
                   ← linha vazia encerra

O programa imprime o tableau de cada iteração (linhas Cj, base com Cb e b,
Zj e Cj−Zj) — requisito 3 — e termina com:

    === Resultado ===
    Status: Otimo
    Solução final? Sim
    Iterações: 2
    Valor ótimo (Z/C): 36
      x1 = 2
      x2 = 6

EXEMPLO COM MINIMIZAÇÃO (Duas Fases)
------------------------------------
    Min C = 4x1 + x2
    s.a.  3x1 +  x2  = 3
          4x1 + 3x2 >= 6
           x1 + 2x2 <= 4

Entrada manual:
    9
    min 4 1
    3 1 = 3
    4 3 >= 6
    1 2 <= 4
    (linha vazia)
(ou simplesmente a opção 7 do menu)
O programa executa a FASE 1 (minimiza a soma das artificiais) e depois a
FASE 2, terminando com C = 3.4, x1 = 0.4, x2 = 1.8.

MENSAGENS DE DIAGNÓSTICO (acidentes do SIMPLEX)
-----------------------------------------------
O campo "Status" do resultado indica:

  Otimo            Solução ótima única encontrada.
  Degenerado       Ótimo atingido, mas houve degeneração (empate no teste
                   da razão θ ou variável básica com valor zero).
                   Ex.: Max 3x1+9x2; x1+4x2<=8; x1+2x2<=4.
  Inviavel        A região viável é vazia: ao fim da Fase 1 restou variável
                   artificial na base com valor positivo.
                   Ex.: Max 3x1+2x2; 2x1+x2<=2; 3x1+4x2>=12.
  Ilimitado        Problema sem fronteira: a coluna pivô não tem coeficiente
                   positivo, a variável pode crescer indefinidamente.
                   Ex.: Max 2x1+x2; x1−x2<=10; 2x1−x2<=40.
  OtimosAlternados Há mais de uma solução ótima: variável não-básica com
                   Cj−Zj = 0 na tabela ótima.
                   Ex.: Max 2x1+4x2; x1+2x2<=5; x1+x2<=4.
  Interrompido     Parada pelo limite de iterações (solução intermediária).

"Solução final? Sim/Não" indica se a resposta é definitiva ou foi
interrompida (requisito 4).
